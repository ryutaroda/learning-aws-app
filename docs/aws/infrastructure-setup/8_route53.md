# Route53設定ドキュメント

## 概要

Route53を使用してカスタムドメインをALBに紐付け、HTTPS通信を実現します。

## 作成順序

### 前提条件

1. ✅ **VPC作成完了**: `learning-stg`
2. ✅ **ALB作成完了**: `learning-alb-stg`

3. [X] **独自ドメイン取得済み**: Route53またはその他のレジストラで取得済み

### 作成手順

#### 1. Route53でドメイン取得

- [X] **ホストゾーン作成**

  - Route53コンソール → ホストゾーン → ホストゾーンを作成
  - ドメイン名: `example.com`
  - タイプ: パブリックホストゾーン
- [X] **ネームサーバーを確認**

  - 作成されたホストゾーンのNSレコードを確認
  - 4つのネームサーバーが表示される

#### 2. Certificate Managerで証明書発行

- [X] **ACM証明書のリクエスト**

  - AWS Certificate Managerコンソールを開く
  - リージョン: `ap-northeast-1`（ALBと同じリージョン）
  - 「証明書をリクエスト」をクリック
  - 証明書タイプ: パブリック証明書をリクエスト
  - ドメイン名: `example.com`（必要に応じて `*.example.com` も追加）
  - 検証方法: DNS検証
- [X] **DNS検証レコードの作成**

  - 証明書のステータスが「検証保留中」になる
  - 「Route 53でレコードを作成」ボタンをクリック
  - Route53に自動的にCNAMEレコードが追加される
  - 数分～数十分で証明書のステータスが「発行済み」に変わる
